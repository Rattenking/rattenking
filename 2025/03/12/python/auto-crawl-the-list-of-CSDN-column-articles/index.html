    1. 场景

    需求就是专栏中文章随着时间写的越多，如果后边需要去查找的时候比较麻烦，比如一些不常用的 git 命令，或者有些开发场景的细节，在之前已经开发完了，现在忘记部分细节，需要在之前的输出文章中去查找，当几十几百篇文章时，查找就比较麻烦，但是如果没发布一篇文章，自己去更新专栏的文章目录又是一个比较繁琐的事情，因此写了一个小的爬取程序，在每次发布新的文章时，运行此程序，就可以更新文章目录，方便后期在需要的时候能够快速查找。

    2. 引入使用模块

    1. requests 获取网页的内容；
    2. re 使用正则匹配文章的发布日期；
    3. time 用于每次获取网页后的等待，防止被 CSDN 识别为爬虫；
    4. datetime 用于文章发布日期的排序格式化；
    5. BeautifulSoup HTML 解析。

    3. 获取专栏文章

    3.1 初始化变量

    1. page 变量初始化为 1，表示从第一页开始获取文章。
    2. articles 列表用于存储所有获取到的文章。
    3. file_name 变量用于存储专栏的名称。

    3.2 循环获取文章

    1. 使用 while True 循环不断获取文章，直到没有更多文章为止。
    2. 构造请求 URL，其中 column_url 是专栏的 ID，page 是当前页码。
    3. 调用 scrape_csdn_blog 函数获取当前页的文章列表，并存储在 current_page_articles 中。

    3.3 检查当前页文章的数量

    1. 如果文章数量少于 40 篇，表示已经获取到了所有文章，跳出循环。
    2. 如果文章数量大于 0，则将当前页的文章添加到 articles 列表中，并增加 page 页码以获取下一页文章。
    3. 如果文章数量为 0，表示出现了异常情况，也跳出循环。
    4. 每次循环结束后，延迟 5 秒，以避免被 CSDN 识别为爬虫。

    3.4 获取文章标题和链接

    1. 调用 get_all_articles_title_and_urls 函数，传入获取到的所有文章列表 articles，该函数会返回一个包含文章标题和链接的列表 markdown_texts。

    3.5 创建 Markdown 文件

    1. 调用 create_markdown_folder 函数，传入当前路径 ‘.’、专栏名称 file_name 和文章标题及链接列表 markdown_texts，该函数会创建一个 Markdown 文件，并将文章标题和链接写入其中。

    3.6 打印完成信息

    1. 打印一条信息，表示专栏更新完成。

    5.3 实现思路

    1. 发送请求：使用 requests 库发送一个GET请求到指定的 column_url，同时设置了请求头 headers 和超时时间 timeout。
    2. 检查请求是否成功：检查HTTP响应的状态码是否为200，表示请求成功。
    3. 解析网页内容：使用 BeautifulSoup 库解析响应的HTML内容。
    4. 提取专栏名称：通过查找HTML中的 h3 标签，并且具有 column_title 类，来提取专栏的名称。
    5. 找到文章标题和链接：先找到具有 column_article_list 类的 ul 标签，然后在这个标签下找到所有的 li 标签，这些 li 标签通常包含了文章的标题和链接。
    6. 返回结果：请求成功，函数返回一个包含专栏名称和文章列表的字典。
    7. 处理请求失败的情况：请求失败（即状态码不是200），函数打印出错误信息，并返回一个包含空专栏名称和空文章列表的字典。

    6.4 实现思路

    1. 初始化列表：创建一个名为 markdown_texts 的空列表，用于存储处理后的文章信息。
    2. 遍历文章列表：使用 for 循环遍历传入的 articles 列表中的每一篇文章。
    3. 提取文章信息：
    • 从当前文章中提取 h2 标签且具有 title 类的文本，作为文章的标题。
    • 从当前文章中提取 a 标签的 href 属性，作为文章的链接。
    • 从当前文章中提取所有 span 标签且具有 status 类的文本，取第一个作为文章的发布日期。
    1. 格式化日期：调用 extract_time_format 函数，将提取到的日期字符串格式化为 年-月-日 时:分:秒 的格式。
    2. 存储文章信息：将提取到的文章标题、链接和格式化后的日期存储到 markdown_texts 列表中。
    3. 排序文章列表：调用 sort_by_date 函数，根据文章的发布日期对 markdown_texts 列表进行排序。
    4. 返回结果：返回处理后的文章信息列表。

    7. 创建 markdown 文件，写入内容

    7.1 实现思路

    1. 循环文章标题和链接重构；
    2. 内容重构：将传入的 markdown_texts 列表转换为一个新的字符串列表 texts。对于列表中的每一篇文章，它提取标题和链接，并将其格式化为一个带有编号的列表项，其中编号使用 zfill(3) 方法进行前导零填充，以确保编号始终是三位数。
    3. 写入文件：使用 with open 语句打开一个文件，文件名为 path 和 foldername 组成的路径，文件扩展名为 .md，表示这是一个 Markdown 文件。文件以写入模式打开，并指定编码为 utf-8，以确保可以正确处理包含非 ASCII 字符的标题。然后，它将重构后的 texts 列表写入文件。如果在写入过程中发生异常，它会捕获异常并打印出错误信息。
