    1. 场景

    在前端项目开发中，图标维护一直是一个比较麻烦的事情：

    1. 第一种：就是直接项目页面里边直接读取图片，这种有个问题，就是如果后期图片地址存在变更，维护就比较麻烦，比如下图就是我 uniapp 项目的图标，如果后期将这些图片传到服务器【微信小程序由于包大小限制，因此图片尽量放在服务器，减小包的大小】，前端必须使用服务器地址，这样是不是直接懵逼。
    2. 第二种：就是为了减小维护成本或者说后期改变的替换成本，就需要一个图标管理的 js 文件文件，将所有图标集中管理，这样后期即便需要替换，直接维护这个 js 文件就可以了。有所进步，但是还是比较麻烦，需要我们人工去维护，每次添加或者修改，都需要开发人员来维护这个文件，比较麻烦，比如改成服务器图片路径，需要替换的成本依然很高。
    3. 第三种：写一个脚本来维护这些图标文件，当然脚本的语言随便你选择，node、python等等，只要可以实现功能，都是可以的，由于我比较熟悉 python 的脚本，所以本文直接以 python 实现自动化图标文件管理。

    1. 递归查询图片文件夹中的所有文件夹和图片文件；
    2. 遍历对应文件夹下的图片文件；
    3. 将图片文件的路径组装为项目中图标 js 文件的字符串；
    4. 创建对应的图标 js 文件。

    4. 获取文件夹下所有文件名称

    4.1 实现思路

    1. 定义函数，接受一个参数 path，表示要遍历的路径。
    2. 调用 get_all_image_names 函数，获取指定路径下的所有图片文件名，并将结果存储在 image_names 变量中。
    3. 如果 image_names 不为空，调用 create_js_file 函数，传入 path 和 image_names 作为参数，该函数的作用是根据图片文件名创建一个 JavaScript 文件。
    4. 获取指定路径下的所有文件和文件夹名，并将结果存储在 names 变量中。
    5. 遍历 names 列表中的每个元素。
    6. 使用 os.path.join 函数将当前路径 path 和当前遍历的文件名 name 拼接起来，形成一个新的路径 new_path。
    7. 调用 check_path_type 函数，检查 new_path 的类型，并将结果存储在 type_name 变量中。
    8. 如果 type_name 等于 ‘dir’，即 new_path 是一个文件夹。
    9. 如果 new_path 是一个文件夹，则递归调用 get_all_file_names 函数，继续遍历该文件夹下的文件和文件夹。

    7.1 代码分析

    1. 定义函数，接受两个参数 path 和 image_names，分别表示要创建 JavaScript 文件的路径和图片文件名列表。
    2. 获取文件名：从 path 中提取文件名，使用 split(‘\‘) 将路径分割成多个部分，然后使用 pop() 方法获取最后一个部分，即文件名。
    3. 从 path 中提取创建路径，使用 split(‘user\‘) 将路径分割成多个部分，然后使用 pop() 方法获取最后一个部分，即创建路径。
    4. 将 create_path 中的反斜杠 \ 替换为正斜杠 /，以确保路径在 JavaScript 文件中正确表示。
    5. 初始化一个空列表 icon_imports，用于存储导入图片文件的 JavaScript 代码。
    6. 定义一个字符串 start_text，表示 JavaScript 文件的开头部分，其中包含导出的常量名称，该名称与文件名相同。
    7. 初始化一个列表 icon_names，用于存储 JavaScript 文件的内容，首先将 start_text 添加到列表中。
    8. 遍历 image_names 列表中的每个图片文件名。
    9. 从图片文件名中提取不带扩展名的部分，作为图标的名称。
    10. 定义一个字符串 icon_name，表示图标的名称，其中包含 V2Icon 后缀，并以逗号和换行符结尾。
    11. 将 icon_name 添加到 icon_names 列表中。
    12. 定义一个字符串 icon_import，表示导入图片文件的 JavaScript 代码【此步骤有一个优势，就是后期如果需要替换服务器路径时，直接将此字符串改为对应的服务器拼接字符串，上传图片都可以自动上传实现】。
    13. 将 icon_import 添加到 icon_imports 列表中。
    14. 将 JavaScript 文件的结束部分 };\n 添加到 icon_names 列表中。
    15. 将 icon_imports 和 icon_names 列表合并成一个列表 js_texts，其中包含了导入图片文件的代码和图标的名称。
