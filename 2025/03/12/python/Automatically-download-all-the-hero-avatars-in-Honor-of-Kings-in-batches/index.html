    1. 前言

    前几天使用 uniapp 实现了一个《扫雷》游戏的小程序，最近又准备使用 uniapp 实现一个《连连看》或者《消消乐》的小游戏，但是在写的时候需要很多图片，由于我本人虽然已经是三十五岁的手残党了，但是没事的时候，还是偶尔会王者几把娱乐娱乐，因此就想到了下载王者的所有英雄头像，来作为实现这个游戏的图片资源，图片只用于自己开发项目，于是就去官网查看现在王者有多少英雄，进行下载，结果去官网一查看，已经120多个英雄了，这要手动一个一个保存，这怎么是一个程序员做出来的事情呢？不是显得我不是一个专业的程序员？一个专业的程序，如果这个事情超过五次需要做相同的操作，我觉得就可以考虑使用脚本来自动帮我们完成这个操作了。

    1. 使用 requests 库发送一个 GET 请求到 herolist_url，并设置了请求头 headers 和超时时间 timeout。
    2. 请求成功，使用 BeautifulSoup 库解析响应的 HTML 内容。
    3. 在解析后的 HTML 中找到 ul 元素，其类名为 herolist clearfix，然后在该元素中找到所有 li 元素。这些 li 元素中包含英雄头像图片。
    4. 打印获取到的头像数量，并返回找到的头像列表。
    5. 请求失败（状态码不是 200），打印错误信息，并返回一个空列表。

    6. 获取头像地址列表

    1. 使用for循环遍历herolist列表中的每个元素。herolist是通过get_header_list函数获取的，包含了页面中所有英雄头像的HTML元素。
    2. 通过heros.find(‘img’)找到当前元素中的标签，然后使用[‘src’]获取该标签的src属性值，即头像的URL。将这个URL添加到header_url_list列表中。
    3. 循环结束后，函数返回包含所有头像URL的列表。

    3. 对于每个头像URL，发送一个GET请求，并设置请求头和超时时间。
    4. 使用正则表达式从头像URL中提取图片名称。

    9. 总结

    1. 下载流程没有问题，图片也能够正常下载。
    2. 问题就是下载的图片和我们在官方看到的少了一部分，查找分析问题原因。

    10. 查找问题

    通过上边代码可以看到头像其实是 js 动态拼接然后插入的，这样就存在一个问题，在我们使用 requests 请求页面的 html 时候，页面还没有渲染完成，所以就出现我们上边的情况，少了部分头像。

    11. 解决办法

    1. 方案一：使用selenium库： 如果你需要更高级的控制，比如等待特定的JavaScript代码执行完毕后再获取页面内容，可以考虑使用selenium库。selenium可以模拟浏览器行为，允许你等待页面加载完成。
    2. 方案二：使用官网的方法，先获取头像列表的配置文件，再解析获取对应头像的 ename， 进行拼接完善头像地址。
