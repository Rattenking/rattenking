    张鑫旭的技术写作指南总结

    读书笔记
    Created At : 2025-03-14 14:30
    Views 👀 :


    不管是觉得是任务，还是兴趣。值得高兴的事情就是我把《技术写作指南》这本小册读完了。当然不是自夸，而是在小册或者CSDN上花钱买课程，可以说估计最少也有近千个打烊了。真真完整看完的，其实不多，小册可以说是第一本。视频的还比较多，我就是那个不怎么容易集中注意力的，很容易就被各种信息干扰，对于这一点我觉得最少值得庆贺，哈哈哈！

    说说我买这本小册的初衷吧！可以说开始是有希望将自己的博客写的更好，必定我也还是一个在CSDN坚持创作了八年的博主，可以说基本还是满足了大佬所说的一年五十篇的要求的。中间也遇到像大佬所说的被骂、不知道写什么等等的问题，我记得当时还因此差不多停更了两年，后来还是被CSDN的一个博主的年终总结所鼓励，才决定继续写下去。所以后来规定平均一周最少发布一篇博客，一个月也就是四篇。因为没有利益，没有动力，说真的是一件很痛苦的事情，可以说一度想要放弃。所以你可以看到这中间我学习了很多，做了很多的学习笔记，其实这也算一种水博客吧！我有印象的就是SQL、opencv、python等。

    为什么学习这些呢？我记得我学习 Java 那会儿全栈比较火，其实那会儿 node 都还是 express 的天下。但是我们公司的后端一直使用的是 Java，所以学习了 node，你就没有实际操作的经验。然后我就学习了 Java 的教程，虽然最后我还是没有转去 Java。为什么呢？还是痛苦导致的，那会儿的 Java 开发工具是 eclipse 和 myeclipse。我印象深刻的是安装一次环境，基本最少一天没有了，中间还需要解决各种报错，这还是在当时公司那些熟手的情况下。所以最后我也就只是能够写一些基础的代码，写一些简单的接口，我记得中间有一段时间我很想转后端，然后我就试着开发了一些项目中实际的接口，比如对接微信和用户地址管理这两个模块。但是由于对那个项目不熟悉，当时调用一个请求第三方接口的工具，在本地和测试环境都没有问题，但是正式上线后，好像就不能使用这个工具，但是我第一次接手，我不知道，然后就让我改，改到凌晨三四点，依然没有找到问题，然后打电话找了大佬看，才知道工具类使用错了。这个事直接将我对 Java 的兴趣打消了，这期间还是不短的时间，也学习了 SQL等。放下了，现在我也就只是能够大概看的懂一些基本的业务代码吧，当然很多年没关注过，可能很多语法都没见过了，这都是常见的事情。

    学习 python 和 opencv 是为了工作的方便，能够快速开发一些小工具。所以就将这些东西做了记录，觉得没有什么深入的东西。可以说学了做了这么多年，基本上都是浮于表面，没有深入研究。当然写的东西也就没有啥深度，为什么我要介意这个问题呢？是因为我最开始从在学校的时候，我的梦想就是写一本自己的书，能够实体书出版。这是我第一次把这个想法写出来，因为就和大佬说的，我觉得我写的那些东西没有什么用，不好意思说出来，也觉得没有梦想实现的那一天。

    但是人总要在某个时候打一下鸡血，所以看到这个小册的时候，我又再次鸡血的买了。然后读了这本小册，其实收获说真的很大，最主要的其实还是实操，就像大佬所说：“宁可花大把的时间和精力做十件平庸的事情，也不愿意做一件极致的事情。”其实可以说，基本上我就是这样的人，做开发快十年了，基本都把时间用在了平庸的事情上。我不知道这个道理吗？其实不是，像稻盛和夫的：你不会的东西，觉得难的东西一定不要躲，先搞明白后精湛，你就比别人优秀了。因为大部分人都不舍得花力气去钻研，自动淘汰，所以你执着的努力就占了大便宜。请相信，相信的力量，选择一件事就坚持去做，低头赶路，不问前程，会被越来越多的人看见，也会被越来越多的人吸引。如果有一天你的努力配得上你的梦想，那你的梦想一定不会辜负你的努力。可以说我在很早就读了，没有做到有什么用？就像程朱理学的格物致知，你不致知，就没有目的。光知道，光去格物，不实际操作，不深入研究，那么写再多的东西，也没有用。

    所以抓住一个方向，深入研究，是很有必要的。学会《技术写作指南》的那些技巧，让你能够走更少的弯路。就像读书，当时你不一定有用，甚至有可能你这一辈子都没有实践书本知识的机会，但是你一定还是要读，要总结，要思考。说不定那天你就用到了。

    就像我写博客，我喜欢把我实现的功能等每一步的实现、想法、逻辑都一步一步的写出来。我记得前段时间就有很多人评论我写的博客一股子 AI 味儿。我开始还会争论，后来慢慢的我就接受了，你说是 AI 就是 AI 吧！又有什么关系呢？我改变不了你的认知，我也不会因为你改变我写作的习惯。为什么要这么写呢？就是内心的自卑吧，害怕表达，害怕被别人看到。就像我的读书笔记，日记（当然不是每天都写，估计平均一个月有一篇吧），我从来不会让除了我以外的人看见，包括我的老婆。不善于表达自己，也不善于展示自己。

    总的来说，其实我个人觉得，能够写的更加深入，肯定是更好的。必定人的精力都是有限的，用你有限的精力去深入学习某方面的知识，是很有必要的。不管是为了养家糊口，还是为了实现自己的梦想。最少目前出本书，可以说对于我来说是奢望。不过，就像大佬说的：“心之所向，自是成长。”

    没有必要在乎一时的得失，其实在你坚持的过程，也是对你自己的一种历练。

    哎，每次脱离代码逻辑的一步一步的完善，我觉得我写的文章就是一坨屎，没有中心，没有主题。算了，不管了，就这样吧！
