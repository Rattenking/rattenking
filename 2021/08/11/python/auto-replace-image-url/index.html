    1. 背景

    由于安全的需要，图片服务器已经变更很多次了，我之前写过【Python 实战】—- 获取所有项目中配置的所有图片路径，这篇博客就是当时图片地址太多，所以需要对前端图标图片进行白名单处理，当时加班实现查询所有项目中图标图片地址的脚本。但是这次坑比的事情来了，服务器和域名还有图片地址的规则全部换了，说到此处，真的是心理有千万只🐎在奔跑，就没有其他事情了吗？天天都搞这种事情，有什么意义？这次没有白名单了，必须所有项目的图片都进行替换，崩溃的一天啊，但是还要写，总不能一个一个图片的处理吧，那估计就真的疯了！

    2. 解决方案

    1. 我写脚本将图片下载到本地，然后再上传的新的域名下，再将新的图片地址替换原来的图片地址；这个方案否决了，因为上传图片需要权限，也就是说上传必须登录，登录为了安全加了图片滑动验证的操作，目前我还没有那个能力解决这个登录问题，因此此方案去掉；
    2. 通过【Python 实战】—- 获取所有项目中配置的所有图片路径将图片地址集中到一个excel中，由后端自己去下载上传这些图片，再将新的图片地址保存到excel原来图片地址后，将新的excel发我，我写脚本进行替换。

    4.1 逻辑分析

    1. wb = openpyxl.load_workbook(file_path)：使用 openpyxl 库加载指定的 Excel 文件。
    2. sheet = wb.active：获取 Excel 文件中的图片地址表。
    3. data = {}：初始化一个空字典，用于存储从 Excel 文件中读取的数据。
    4. 遍历工作表中的每一行数据，从第二行开始（min_row=2），读取前三列数据（max_col=3），并且只读取单元格的值（values_only=True）。
    5. 检查当前行的第一列数据（row[0]）是否已经存在于字典 data 中。
    6. 如果当前行的第一列数据不存在于字典 data 中，则在字典中创建一个新的键值对，键为当前行的第一列数据，值为一个空列表。
    7. 将当前行的第二列和第三列数据（row[1:]）添加到对应键的列表中，函数返回读取并组织好的数据字典。
    8. 注意：此处主要是以图片所在的文件对数据进行分类处理。

    5. 读取对应文件夹替换

    5.1 逻辑分析

    1. 遍历字典 data 中的每个键值对。key 是文件名，value 是一个列表，列表中包含了要替换的字符串。
    2. 使用 with 语句打开文件，文件路径是 base_path 加上文件名 key。’r’ 表示以只读模式打开文件，encoding=’utf-8’ 表示文件的编码格式是 UTF-8。
    3. 读取文件的内容，并将其存储在变量 content 中。
    4. 调用 replace_all_excel_info 函数，将文件内容 content 和要替换的字符串列表 value 作为参数传递给它。replace_all_excel_info 函数会遍历列表中的每个元素，将文件内容中的所有匹配的字符串替换为新的字符串，并返回替换后的新内容。
    5. 使用 with 语句打开文件，文件路径是 base_path 加上文件名 key。’w’ 表示以写入模式打开文件，如果文件不存在，则创建一个新文件。
    6. 将替换后的新内容 new_content 写入到文件中。
    7. 遍历传入的data列表，这个列表中的每个元素都是一个元组，包含两个字符串，第一个字符串是需要被替换的旧字符串，第二个字符串是替换后的新字符串。
    8. 对于data列表中的每个元组item，检查item[0]（旧字符串）是否存在于content（文本内容）中，并且item[1]（新字符串）是否不为空。
    9. 如果上述两个条件都满足，那么就使用replace方法将content中的item[0]替换为item[1]。replace方法会返回一个新的字符串，其中所有匹配的旧字符串都被替换为新字符串。返回新的字符串。
